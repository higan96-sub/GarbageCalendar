# How to Build
```
$ git clone https://github.com/higan96/GarbageCalendar
$ cd GarbageCalendar
$ carthage update --platform iOS
```

# 概要

## アプリ名
- シンプルなゴミの日カレンダーアプリ 「ゴミノヒ」

## コンセプト
- 今日のゴミの日が一目でわかるアプリ

## 設計など
- WebAPIからのデータ取得を行う（想定。サーバーサイドは未着手。）
  - そのため「ゴミの日」のデータのロジック（第二月曜日は燃えないゴミ、など）はアプリ側に持たせず、WebAPIのサーバー側で行うような作りにしている。
  - JSONで1年分のデータを処理しているのはそのため
  - 他の自治体への対応を容易にするという理由もある
  - WebAPIについては今後実装を行うとして、ひとまずアプリのパッケージにJSONデータを含めた形で先行してiOS版のリリースを行う予定
- MVP（Passive View）
  - 特にデータの永続化も必要ないビューアーアプリなのでMVPを採用
  - 依存するライブラリもないのも決めて
  - ModelとPresenterについてのテストもしたかったので
- ライブラリ管理はCarthageを使用
  - 普及度的にはCocoapodが優位だが、ビルド速度の点でこちらを選択

## デザインなど

- アイコン、ゴミの日リスト（ホーム画面）、ゴミの日カレンダー画面など、「オレンジの枠で囲ったものが重要（今日のゴミ）」というメッセージで統一している
- 広い世代が使うアプリを想定しているので、文字と画像は可能な限り大きくしている。
